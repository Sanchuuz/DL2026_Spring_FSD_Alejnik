# Часть 1. Проектирование и анализ

## 1.1. Пользовательские сценарии
* **Поиск по городу**: «Как пользователь, я хочу ввести название своего города, чтобы мгновенно увидеть текущую температуру и получить мем, который точно описывает мои ощущения от погоды за окном».
* **Автоопределение**: «Как пользователь, я хочу разрешить доступ к геолокации, чтобы приложение само подсказало, стоит ли мне сегодня брать зонт, подкрепив это забавной картинкой».
* **Шеринг настроения**: «Как пользователь, я хочу нажать одну кнопку, чтобы отправить «погодную открытку» с мемом другу в Telegram, разделив с ним иронию над плохой погодой».

## 1.2. Функциональные требования
**Обязательные:**
* Интеграция с внешним API погоды (например, OpenWeatherMap).
* База мемов, классифицированная по погодным условиям (ясно, дождь, снег, жара, холод).
* Отображение основных данных: температура, описание погоды, иконка и соответствующий мем.
* Поиск погоды по названию города.

**Опциональные:**
* Определение местоположения по IP или GPS.
* Возможность «реролла» мема (показать другой мем из той же категории)
* Кнопка копирования ссылки или генерация изображения для соцсетей.
* Админ-панель для загрузки новых мемов в базу.

## 1.3. Проектирование API
* **GET `/api/weather?city=Minsk`**
    * Параметры: `city` (string).
    * Ожидаемый ответ: JSON с объектом, содержащим температуру, описание погоды (например, "дождь") и URL подходящего мема.

## 1.4. Модель данных
* **Сущность Meme**:
    * `_id` (ObjectId): уникальный идентификатор записи.
    * `imageUrl` (String): ссылка на изображение мема.
    * `weatherType` (String): категория погоды: hot, cold, rain, snow, windy.
    * `text` (String): забавный текст или шутка (по желанию).
    * `tags` (Array): дополнительные метки для более точного поиска.

## 1.5. Ключевые технические решения
* **Стек**: React (Frontend), Node.js (Backend), MongoDB (Database), OpenWeatherMap API, Axios.
* **Обоснование**: React — идеален для динамического обновления превью без перезагрузки страницы, Node.js (Express) — легковесный и производительный вариант для создания REST API, MongoDB — гибкая документная модель отлично подходит для хранения объектов «мем» с разными тегами и метаданными, OpenWeatherMap API - для получения актуальных данных о погоде в реальном времени, Axios - библиотека для выполнения HTTP-запросов как на фронтенде, так и на бэкенде.
